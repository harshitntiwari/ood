Steps for object-oriented design problem solving:

1. Requirements: Think through and fix the core requirements.
2. Entities: What will be the core entities in your design? You can generally figure out these entities by looking at the nouns in your requirements.
3. Class design: Start designing the classes based on the entities. Write down the state (attributes) and behaviour (methods) for these classes.
4. Implementation: 
  - objectives of implementation should be to code: primary capabilities + error handling. 
  - Implement helper functionalities (whichever applicable) like: enums, strategies, factories, observers etc.
  - Implement core logic of the core entities.
  - Finally handle edge cases (error handling).
